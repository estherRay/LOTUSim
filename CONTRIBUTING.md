# Contributing to LOTUSim

LOTUSim supports research, experimentation, and interoperability in multi-agent simulation. This document explains how to contribute. All contributions must be under EPL2.0 license.

## Governance Committee

The LOTUSim Governance Committee, based at Naval Group, oversees the project's overall direction. The committee:

- Publishes a bi-annual roadmap, highlighting priority topics and features
- Reviews, accepts, and integrates community contributions
- Publishes a new release every 6 months with the validated contributions

The committee may include external members in the future, such as key partners or major contributors.

## Roadmap

The roadmap is based on:

- Naval Group's internal needs
- Partner priorities
- Community contributions

For roadmap questions, please contact [our support](lotusim_support@naval-group.com).

## How to Contribute

You can contribute in several ways:

- **New features** – e.g. new algorithmic modules for platform guidance
- **New models** – for platforms, sensors, or environments
- **Enhancements** – improving existing functions, models, documentation
- **Data contributions** – use cases, simulation scenarios, or datasets
- **Bug fixes** – code corrections, refactoring, or maintainability improvements

### Contribution process

1. **Open an Issue**

    - If none exists, open one on GitHub
    - Use one of the following labels: new_feature, new_model, enhancement, data, or bugfix
    - Clearly describe your proposal, including its objectives, use cases, scientific background, and references (if relevant)
    - Specify how your contribution aligns with the LOTUSim roadmap

2. **Announce Your Work**

    - Comment on the issue to say you're working on it, with a short description of your approach

3. **Fork and build**

    - Fork the LOTUSim repository and implement your changes

4. **Test and Validate**

    - Ensure your code passes all tests and meets projects requirements
    - A dedicated testing guide will be available soon in the README

5. **Submit a Pull Request**

    - Open a pull request referencing the related issue
    - Clearly explain the purpose and scope of your changes

## Pull Request Review and integration

Depending on its type, a pull request is either:

- Accepted directly by the maintainers (expected feedback within 1 month), or
- Escalated to the LOTUSim Governance Committee (feedback within 6 months)

If not accepted right away, you may get feedback asking for clarification, documentation, or code changes. Once approved, your contribution is merged into the main branch and included in the next release.
